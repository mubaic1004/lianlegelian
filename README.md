# 炒了个菜 🍳

连连看 × 菜谱合成 × 顾客点单的可爱风网页小游戏(中/英双语,瓜皮工作室出品,当前 **v1.0.0.4**):32 种食材、10 道菜谱、5 个关卡,难度从"30 秒通关"一路爬到"通过率 <1% 的传说灶神"——且每一局都保证有解。

🎮 在线玩:https://mubaic1004.github.io/lianlegelian/

## 玩法

- **顾客点单(过关目标)**:顶部点单栏是本局顾客的订单(🐰 要蛋炒饭×2……),**清空牌面 + 出齐所有订单**才算过关。同款配对会吃掉菜谱食材,记得给顾客留料。
- **同款配对**(金光):两张相同食材、两折以内连得通,直接消除。
- **菜谱合成**(绿光):菜谱搭子连到一起会合成新菜——鸡蛋+米饭=蛋炒饭、玉米+黄油=爆米花……共 10 道菜;**每出 3 道菜奖励 1 个随机道具**。
- **备菜槽与新鲜度**:连不到的食材点一下进槽(有可连同伴时需再点一下自己),槽内同款或搭子相遇即消;塞入第 7 张单牌失败。**槽内食材每走一步掉 1 格新鲜度(牌底绿条),归零变质直接判负**——「弹出」可把最老的 3 张送回牌面并重置新鲜度。
- **道具**(每局各 1,可靠出菜攒):撤回(失败时可当复活)、洗牌、弹出。
- 连线飞在两张牌较高层之上,只被不低于该层的牌挡住——挖得越深越难连。

## 关卡与难度(bot 实测通过率)

| 关卡 | 牌数 | 食材 | 点单 | 保鲜 | bot 通过率 |
|---|---|---|---|---|---|
| 1 开火热灶 | 24 | 6(3 菜谱) | 2 | 24 步 | ~100% |
| 2 小试牛刀 | 78 | 14(4 菜谱+6 散装) | 3 | 22 步 | 55%~72% |
| 3 渐入佳境 | 126 | 20(4 菜谱+12 散装) | 4 | 20 步 | 10%~16% |
| 4 地狱后厨 | 152 | 22(6 菜谱+10 散装) | 5 | 18 步 | 1%~4% |
| 5 传说灶神 | 188 | 28(8 菜谱+12 散装) | 6 | 19 步 | **0.3%~1.7%** |

难度核心设计:菜谱搭子相当于跨类配对(让人上头也让局面更活);**散装食材**(无搭子)占比、**保鲜步数**和**订单数**是三个难度旋钮。校准 bot 会照顾订单(避免把点单食材同款消光)并优先抢救快变质的槽内牌。

## 本地运行

纯静态页面,任意 HTTP 服务器即可(ES Module 需要走 http):

```bash
python3 -m http.server 8931
# 打开 http://localhost:8931
```

## 文件结构

- `engine.js` — 纯逻辑引擎(无 DOM,浏览器 / Node 通用):食材/菜谱表、布局生成、遮挡判定、连连看 BFS、卡槽状态机、校准 bot
- `ui.js` — 渲染与交互:双色发光提示、连线/合成动画、出品栏、道具、结算、音效(WebAudio 合成)
- `style.css` — 黏土拟物(claymorphism)粉彩风格
- `sim.mjs` — 难度模拟器

## 难度是怎么校准的

每关的布局种子都经启发式 bot(懂菜谱合成)批量试玩筛选:

```bash
node sim.mjs 5 1 60 300   # 关卡 5,种子 1~60,每个 300 局
```

入选标准按关卡通过率带宽(见上表),**至少赢 1 局即确认该布局有解**。种子池在 `engine.js` 各关的 `seeds`,开局随机抽取。调难度旋钮:`typeList`(菜谱/散装配比)、`layout()`(层数、盲堆深度),改完重跑 sim 回填种子即可。
